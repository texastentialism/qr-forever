# CLAUDE.md — QR Forever

## What this is
A QR code generator for print media. Core constraint: **QR codes must work forever**. Sister's business (Porchlight Studios) prints these on physical advertisements, so there can be no dependency on external redirect services that could disappear.

## Core architectural rule
**The QR code encodes the destination URL directly.** Never route through a shortener, redirect service, or database lookup. This is the entire product's value proposition. Violating it breaks the promise.

- No `/r/[slug]` routes
- No database writes at QR creation time
- No third-party tracking embedded in the encoded URL
- Client-side generation only (no server round-trip)

If Phase 2 tracked-links are ever added, they must be clearly labeled as a DIFFERENT product mode — opt-in, disclosed tradeoff ("this QR depends on our servers staying online").

## Tech stack
- Next.js 16 (App Router, Turbopack, server components by default)
- TypeScript
- Tailwind CSS v4
- `qr-code-styling` — the QR generation library
- `lucide-react` — icons
- Deployed to Vercel (auto-deploy from `main`)

## Key files
- `src/app/page.tsx` — landing/marketing wrapper + renders `<Generator />`
- `src/app/layout.tsx` — root layout, fonts, metadata
- `src/components/Generator.tsx` — the entire QR generator (URL input, presets, fine-tune, live preview, downloads)

## Conventions
- All interactive UI in `src/components/` as client components (`"use client"`)
- Keep the generator as one cohesive component for now — do not pre-split until it exceeds ~800 lines or has clear seams
- Feature branch + PR for any non-trivial change (Tyler's global rule)
- Never invent preset colors for "Porchlight" without asking — the warm/sage/indigo presets are generic starting points

## Users
- **Tyler's sister** — Porchlight Studios, primary user, prints QRs on ads
- **Tyler** — may use for his own projects, may expand to broader tool eventually
- **Anyone** — it's public, free, no accounts

## What NOT to do
- Do not add analytics trackers to the encoded URL (e.g., appending `?utm_source=qr`). If we want analytics, it's the destination site's job, not ours.
- Do not add signup/accounts. The whole point is frictionless + trust.
- Do not add a "generated by QR Forever" watermark or embed any URL that points back to us.
- Do not introduce dependencies that require API keys to build (keeps Vercel deploys trivial).

## Phase 2 (not built)
Optional tracked-link mode. If/when added:
- Clearly labeled as a separate tool / page (`/tracked`)
- Uses Vercel Postgres (free tier) for link table + clicks table
- Redirect route logs click server-side, redirects
- Export-to-static-redirects capability so users can migrate OFF our infra if we ever sunset it
- Disclosed in UI: "this mode depends on qrforever.vercel.app staying online"
